Used Dataset

Kaggle Fake and Real News Dataset
by Cl√©ment Bisaillon

This dataset has True.csv and Fake.csv files wich contains True news and Fake news respectively. Both files has around 22000 of Unique News and 4 Attributes (i.e. Title, Text, Subject and Date). Combined size of both csv is 110.98 MB.

Working-Flow

STEP 1 - Data Acquisition

Downloaded the dataset from Kaggle, extracted the dataset.

STEP 2 - Data Loading

Loaded this dataset to Jupyter Notebook as a data frame (here in CSV format) using the Pandas module.

STEP 3 - Data Preprocessing

Performed certain data preprocessing operations to remove redundancy and get homogeneity.

STEP 4 - Data Cleaning

Cleaned text data using Regex and some NLP methods like Lemmatization to get uniform and semantically meaningful text data for the next steps.

STEP 5 - Feature Extraction

Kept necessary features and labels by dropping unnecessary data fields.

STEP 6 - Train-Test splitting

Dataset split into training and testing features and labels.

STEP 7 - Data Vectorizing

Text data can't be directly processed by ML models, So it is necessary to convert news text into vectors. Here TFIDF(Term Frequency Inverse Document Frequency) technique was used to vectorize data.

STEP 8 - Model Building

Imported Logistic Regression, Decision Tree Classifier, Gradient Boosting Classifier, and Random Forest Classifier models from Sklearn module and built model by instantiating the object.

STEP 9 - Training & Testing

Trained and Tested previously instantiated model. Also measured various metrics like Accuracy, Precision, Recall, F1 score, Confusion Matrix, etc.

STEP 10 - Pickle & Serialization

Serialized the entire ML pipeline in the Pickle file by Joblib module so trained and tested models can be available to use anytime for predicting new data.

STEP 11 - Model Deployment

Created a web app using Flask as back-end and HTML, CSS as front-end stack. Connected web app with serialized ML models and deployed on Heroku
